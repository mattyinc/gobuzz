Your first Go'Buzz website edit.

Follow one step at a time. By the end, you will have the website running on your laptop, your own teaser-page change on GitHub, and a private Vercel preview ready for review.

Before you begin

Matty needs to complete these owner-only actions once. Everything else is yours.

  • 1. GitHub accessInvite you as a collaborator on mattyinc/gobuzz.
  • 2. Accept the emailYou must accept GitHub's invitation before your first push.
  • 3. Vercel is automaticYou do not need to install Vercel. GitHub creates the preview and production deployments.
Which laptop are you using?The installation instructions will adapt to your choice.
Complete these in order

    Step 1 of 8

    Get your accounts ready.

    GitHub stores the shared code. VS Code is where you edit it. Vercel turns approved GitHub changes into the live website.

    1. Create or sign in to GitHub

      Open GitHub and create a free account if you do not already have one. Use an email address you check regularly.

      Open GitHub sign-up

    2. Accept Matty's invitation

      Look for an email from GitHub. Open it and choose View invitation, then Accept invitation.

      Checkpoint: while signed in, you can open github.com/mattyinc/gobuzz.

    3. Know the safety rule

      Never paste passwords, private keys, or the contents of .env.local into GitHub. The project already tells Git to ignore that file.

    Step 2 of 8

    Install Git.

    Git remembers every change and lets both of you work on the same website without emailing files back and forth.

    1. Download Git for Windows

      Use the official download. Open the downloaded installer and keep the recommended choices by pressing Next. Choose Install at the end.

      Download Git for Windows

    2. Restart VS Code

      Close every VS Code window and open it again. This lets VS Code find the new Git installation.

    1. Check whether Git is already available

      In VS Code, choose Terminal → New Terminal. Run the command below. If macOS asks to install developer tools, approve it and wait for the installation.

      git --version

    2. Use the official installer if needed

      If the first method does not work, open Git's official Mac download page and follow one of its listed installation choices.

      Git for macOS

    1. Confirm it works

      Open Terminal → New Terminal in VS Code and run:

      git --version
      Success looks like: git version 2.x.x. Your numbers may be different.

    2. Tell Git who you are

      Replace the examples with your own name and the email used for GitHub. Run each line separately.

      git config --global user.name "Your Name"
git config --global user.email "you@example.com"

    Step 3 of 8

    Install Node.js.

    Node.js runs the Go'Buzz website on your laptop. npm comes with it and downloads the project pieces the website needs.

    1. Download the LTS version

      Open the official Node.js page and choose the version marked LTS. LTS means the stable version recommended for most people.

      Download Node.js LTS

    2. Run the installer

      Accept the standard choices. When it finishes, close and reopen VS Code.

    3. Confirm Node and npm

      Open a new VS Code terminal. Run both commands:

      node --version
npm --version
      Success: each command prints a number beginning with v or a version number. Red error text means you should restart VS Code and try again.

    Step 4 of 8

    Download the website from GitHub.

    In Git language, downloading a working copy for the first time is called “cloning.” You only clone once.

    1. Choose a normal projects folder

      Good choices are Documents or a folder called Projects. Avoid OneDrive, iCloud Drive, Dropbox, and Google Drive folders because syncing can confuse Git.

    2. Clone in VS Code

      Press Ctrl + Shift + P on Windows or Command + Shift + P on Mac. Type Git: Clone, select it, and paste this address:

      https://github.com/mattyinc/gobuzz.git

      Choose your projects folder. When VS Code asks whether to open the cloned repository, choose Open. If asked whether you trust the authors, choose Yes.

    3. Sign in when GitHub asks

      Your browser may open. Sign in to GitHub and authorize VS Code. Return to VS Code when finished.

    4. Confirm the right folder is open

      The Explorer on the left should show src, public, and package.json.

    Step 5 of 8

    Run Go'Buzz on your laptop.

    This creates a private local copy at localhost. Only you can see it, and changing it does not affect the live website.

    1. Download the project dependencies

      In VS Code, choose Terminal → New Terminal. Confirm the terminal path ends in gobuzz, then run:

      npm install

      This can take several minutes the first time. Warnings are usually fine; wait until the terminal lets you type again.

    2. Start the local website

      npm run dev
      Keep this terminal open. Closing it stops the local website. To stop it deliberately, click the terminal and press Ctrl + C.

    3. Open the teaser page

      Hold Ctrl or Command and click the terminal's localhost link, or open this address in your browser:

      Open local teaser page
      Checkpoint: you can see “Move. Think. Recover.” and the current sauna section.

    Step 6 of 8

    Make your teaser-page change.

    Create your own branch first. A branch is a private lane for your edit, so the live website stays untouched while you work.

    1. Create a branch

      Open a second terminal using the + button in the terminal area. Run:

      git switch -c brother/teaser-update
      Use a new name next time: for example brother/change-headline or brother/new-photo.

    2. Open the teaser file

      In the left Explorer, open these folders in order: src → app → teaser, then open page.tsx.

      src/app/teaser/page.tsxWords, sections, and the sauna image
      src/components/hero.tsxMove / Think / Recover hero
      public/facilities/Sauna and facility image files

    3. Find the exact section

      With page.tsx open, press Ctrl + F or Command + F. Search for:

      Something quieter is taking shape.

      Edit only the words between the opening and closing tags. Be careful not to remove characters such as <, >, braces, or quotation marks around code.

    4. Save and inspect

      Press Ctrl + S or Command + S. Your browser should update automatically. Check the page on a narrow browser window too, so long text does not overflow.

      If the page turns red: undo your last edit with Ctrl/Command + Z. Read the first error, or ask Matty before changing more code.

    Step 7 of 8

    Save the change to GitHub.

    Saving in VS Code writes the file on your laptop. These Git commands record the change and send your branch to GitHub.

    1. See what changed

      git status

      You should see src/app/teaser/page.tsx listed as modified.

    2. Run the safety checks

      npm run lint
npm run build

      Run one line at a time. Continue only when both finish without an error. The build can take a minute.

    3. Record your edit

      git add src/app/teaser/page.tsx
git commit -m "Update teaser sauna section"

    4. Push your branch

      git push -u origin brother/teaser-update

      If a browser opens, authorize GitHub. Refresh the repository page afterward.

    Step 8 of 8

    Review, merge, and deploy.

    GitHub and Vercel now do the heavy lifting. Your branch gets a preview first; production changes only after the pull request is merged.

    1. Open a pull request

      On the GitHub repository, choose Compare & pull request. Write a short summary of what you changed, then choose Create pull request.

      Open Go'Buzz pull requests

    2. Wait for the Vercel preview

      Vercel automatically builds your branch. In the pull request, wait for the Vercel check to finish, then open its Preview link. Check desktop and mobile before merging.

      A preview is not the live website. It is a safe, shareable copy for reviewing your exact branch.

    3. Ask Matty to review and merge

      When both of you approve the preview, merge the pull request into main. The connected Vercel project automatically deploys main to production.

    4. Start clean next time

      Before every new edit, return to main, download the latest work, and create a fresh branch:

      git switch main
git pull
git switch -c brother/describe-the-next-change
      You did it. You edited the real project, reviewed it safely, and used the same workflow professional teams use.

    When something does not work

    “git” or “node” is not recognized

    Close every VS Code window, reopen it, and open a new terminal. If the problem remains, restart the laptop. Then rerun git --version or node --version.

    Windows says npm scripts are disabled

    In VS Code's Terminal panel, open the profile menu beside the + button, choose Select Default Profile, select Command Prompt, and open a new terminal. Run the npm command there.

    Port 3000 is already in use

    The terminal may automatically offer localhost:3001. Use that address. Or stop the older server by clicking its terminal and pressing Ctrl + C.

    The invitation form does not save locally

    Visual teaser edits do not need private credentials. Full Supabase testing requires a local environment file from Matty. Share it through a password manager, never GitHub, and name it exactly .env.local.

    Git reports a merge conflict

    Stop and ask Matty for help. Do not delete unfamiliar code or force the push. Your work is still saved on your branch.

    I am afraid I changed the live website

    Changes on your laptop and your brother/... branch are not production. The live website changes only after the pull request is merged into main and Vercel finishes its production deployment.
